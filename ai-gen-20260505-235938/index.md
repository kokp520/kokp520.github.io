# GraphRAG：結合知識圖譜與 LLM 的深度檢索增強生成新範式

<!--more-->

在生成式 AI 的應用開發中，**RAG (Retrieval-Augmented Generation)** 已經成為了解決大語言模型 (LLM) 幻覺問題、提供即時私有數據支持的標準方案。然而，隨著應用場景從簡單的「查文檔」演進到「複雜分析」，傳統的 **Baseline RAG**（基於向量相似度的檢索）開始顯露其局限性。

今天我們來探討由微軟研究院提出的 **GraphRAG** 技術，這是一種將 **知識圖譜 (Knowledge Graph, KG)** 與 LLM 深度結合的檢索增強生成新範式，它如何重新定義了我們對海量非結構化數據的理解與檢索。

## 1. 傳統 RAG 的痛點：迷失在向量空間中

傳統的 RAG 依賴於將文本切割成 **Chunks**（片段），然後通過 **Embedding**（嵌入）模型轉化為向量。當用戶提問時，系統尋找語義最接近的片段。

這種方法在處理「局部」資訊時表現優異，但在以下場景中往往會失效：
*   **全局性問題 (Global Queries):** 例如「這份 100 頁的調查報告中，最核心的技術爭議點有哪些？」向量檢索無法掃描全域，只能抓取幾個片段。
*   **跨領域關聯 (Cross-domain Linking):** 當資訊散落在文件的不同章節，且缺乏直接語義重合時，單純的餘弦相似度 (Cosine Similarity) 很難建立連結。
*   **主題聚合 (Thematic Aggregation):** 難以針對某一特定實體（Entity）進行全方位的總結。

## 2. 什麼是 GraphRAG？

**GraphRAG** 的核心思想是：**先用 LLM 將非結構化文本轉化為結構化的知識圖譜，再利用圖的結構進行層次化摘要與檢索。**

它不再只是簡單的「切片與檢索」，而是一個包含多個階段的數據加工流水線：

### A. 實體與關係提取 (Indexing Phase)
系統遍歷所有文本，利用 LLM 識別出其中的 **Entities**（如人名、公司、概念）以及它們之間的 **Relationships**（關係）。這一步將雜亂的文本變成了點與線構成的網。

### B. 社群偵測 (Community Detection)
這是 GraphRAG 最天才的地方。它利用圖論中的算法（如 **Leiden Algorithm**），將緊密相連的實體劃分為不同的 **Communities**（社群）。例如，在一本小說中，某個家族的成員會自動聚集成一個社群。

### C. 層次化摘要 (Hierarchical Summarization)
LLM 會為每個社群生成摘要。從小社群到大社群，形成一個金字塔結構。這意味著，當你問全局問題時，系統可以直接查閱高層級的社群摘要，而不需要閱讀每一個字。

## 3. 檢索模式：Global vs. Local Search

GraphRAG 提供了兩種主要的檢索模式，應對不同的需求：

*   **Global Search (全局搜尋):** 
    *   **場景:** 總結、趨勢分析、宏觀對比。
    *   **機制:** 檢索預先生成的社群摘要，將其作為 Context 餵給 LLM。這解決了 Context Window 限制下無法處理全量數據的問題。
    *   **關鍵詞:** *Map-Reduce*（分片處理再合併）。

*   **Local Search (局部搜尋):** 
    *   **場景:** 特定實體的細節問答。
    *   **機制:** 結合向量檢索與圖關聯。系統會找到與問題相關的實體，並拉取其鄰近節點（Node）與關係（Edge）的資訊，提供比純向量更豐富的背景。

## 4. 技術架構中的挑戰與工程考量

雖然 GraphRAG 效果驚人，但在落地時需要考慮 **Cost (成本)** 與 **Latency (延遲)**：

1.  **Index 成本極高:** 在建立索引階段，需要耗費大量的 LLM Tokens 來提取實體與生成摘要。對於 100 萬字的文件，可能需要數百次 API Call。
2.  **動態更新難度:** 傳統 RAG 只需要新增向量，而 GraphRAG 需要重新平衡圖結構或更新社群摘要，這在數據頻繁變動的場景中是個挑戰。
3.  **Graph DB 的選擇:** 雖然可以使用像 Neo4j 這樣的傳統圖數據庫，但目前的 GraphRAG 實作更多傾向於在內存中處理或存儲為 Parquet 格式，這取決於數據規模。

## 5. 實踐建議：Hybrid RAG 是未來

對於大多數工程師來說，並非所有場景都需要 GraphRAG。最佳實踐通常是 **Hybrid RAG (混合檢索)**：
*   使用 **Vector Search** 處理簡單、精確的查找。
*   引入 **BM25**（傳統關鍵字搜尋）輔助。
*   針對需要「深度思考」或「總結全貌」的部分，引入 **GraphRAG** 的社群摘要機制。

## 結語

GraphRAG 的出現，標誌著 RAG 技術從「語義相似度」向「結構化理解」的重大飛躍。它賦予了 AI 一種更接近人類的閱讀方式：先看整體框架，再深入細節，最後建立關聯。

隨著推理成本 (Inference Cost) 的持續下降，我們有理由相信，具備「圖譜思維」的 AI 應用將成為企業級 AI 知識庫的標配。
